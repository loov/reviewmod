## Dependency Review Checklist
Look for:
- Usage of deprecated functions or types
- Functions marked as unsafe or experimental
- Importing internal packages from external modules
- Circular dependency potential
- Heavy dependencies for simple operations
- Vendor-specific code in generic packages
- Platform-specific code without build tags
- Importing test packages in non-test files
- Using reflect when type-safe alternatives exist
- Importing packages only for side effects without comment
- Violations of intended package layering
- Using cgo when pure Go alternatives exist

Only report actual dependency issues. If no issues found, return {"issues": []}.
